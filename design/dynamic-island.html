Claude Dot · Requirements v0.3 · 2026-06-03

The Dynamic Island for Claude Code sessions

把菜单栏猫头鹰的「聚合状态」搬到 MacBook 刘海周围， 形成一个常驻可见的环境层。折叠态只显示三件事： 图标 · 数字 · 优先级最高的状态文字；悬停展开为非空闲会话列表。 所有视觉沿用 design/DESIGN.md，岛体始终为纯黑以与刘海无缝衔接。

0 · 背景 · 目标 · 非目标

背景。 当前 Claude Dot 通过菜单栏 owl glyph 暴露聚合状态，需要点击展开 popover 才能看到细节。 对于「同时跑多个 session」的重度用户，菜单栏图标信息密度太低 —— 看不到 activeCount、看不到正在等待的工具、看不到错误摘要。

目标。

  • 常驻可见：无需点击即可获知活跃会话数与最高优先级状态。
  • 零干扰：idle 时右段隐藏、整体收窄到仅 owl + 物理 notch 占位的悬浮药丸， 不抢视觉焦点。
  • 零下探（常驻态）：closed 态高度 ≤ 菜单栏高度，仅在左右方向延伸把 notch 包成一条横向加长胶囊 —— 参考高德地图的灵动岛形态。不允许侵入下方内容区，避免遮挡应用界面。 展开态（opened，由用户主动或事件触发）仍允许下探，详见 §2 与 §4。
  • 一跳直达：展开后点击行直接 jump 到会话所在终端。
  • 零新数据源:复用 popover 已有的 SessionVM 与 refresh() tick。
  • 视觉一致：与 popover、菜单栏 owl 同色、同节拍、同 priority 函数。

非目标。

  • 不在岛上代答权限审批：approvalCard 渲染请求 + 提供 jump， 真正的 yes/no 仍在终端键入（Claude Code 的 permission prompt 是终端 UI，岛体不远端代答）。
  • 不展示 token 用量百分比、subscription limit —— 那是 popover 的英雄区域，岛上不重复。
  • 不替代菜单栏 owl —— 两者并存，受控于独立开关。
  • 不支持外接显示器复制岛体 —— 仅显示在内建屏。
  • 不引入网络栈（不做 Watch / Bonjour / SSE 等，留给独立模块）。

2 · 功能需求 · 折叠态三要素

2.0 · 形态与占位约束（resting footprint）。 折叠态是一颗悬浮在菜单栏里的全圆角横向加长胶囊（药丸） —— 形如 iOS Dynamic Island 的横向版本 / 高德地图灵动岛。

  • 高度（运行时算，禁止硬编码）：closed 态总高 = 菜单栏可视高度 - 2pt （上下各 1pt 气口），必须从 NSScreen 实时算，不能写死 28pt。 实测值：14"/16" MBP 含刘海 ≈ 38pt → 岛高 36pt；M1 Air 非刘海 ≈ 24pt → 岛高 22pt； 外接屏断连后菜单栏也可能跳到不同高度。算法见 §9 "菜单栏高度差异"。 禁止下边缘超过菜单栏底线一像素 —— 不能遮挡下方窗口内容。 禁止明显矮于状态栏图标 —— 视觉上必须与系统 status item（如微信、电池）等高同行。
  • 圆角：四角全圆，radius = height / 2（真胶囊 / 药丸形态）。 高度变时圆角必须同步重算，不能写死 14pt。 不再采用"顶部直角贴屏顶 + 底部圆角"的早期方案 —— 视觉上是悬浮药丸而非屏顶延伸。
  • 水平形态：以物理 notch 区域为中心，向左右两段对称延伸出信息区； 整体黑色 #0A0A0A 与 notch 同色，视觉上是"包裹住 notch 的悬浮黑色药丸"。
  • notch-core 宽度必须运行时算，禁止硬编码 180pt。不同机型物理 notch 宽差异显著 （14" Pro ~205pt / 16" Max ~225pt），固定 180pt 会让 trail 的首 token（数字）落进物理刘海 被挡住。算法： notchCoreWidth = max(auxiliaryTopLeftArea.frame.maxX - auxiliaryTopRightArea.frame.minX, safeAreaInsets.top * 5) + 24pt 安全余量 （左右各 12pt 缓冲，避开 notch 的边缘渐变）。fallback 到 220pt 而非 180pt。
  • 左段（lead）：仅一个彩色换脸 owl glyph（22×22，glyph 自带状态色）。 不放状态脉冲点 —— owl 的颜色本身就是状态（run 绿 / wait 黄 / err 红怒 / idle 炭灰）， 再叠一个 .pulse 是冗余。禁止放文字。 （脉冲点仅在展开的 sessionList 行内、每个 task 自己的状态点保留。）
  • 右段（trail）：仅两个 token —— {activeCount} {状态单词}，如 "2 Running" / "1 Awaiting" / "1 Error"。 状态单词按 §3 优先级取最高级的单词（Error / Awaiting / Running / idle 隐藏整段）。 不要展开标题、tool 名、cwd 等长文案 —— 那些信息留给点击展开后的列表/卡片， 岛上常驻态只做"一眼可读"的状态摘要。 禁止 marquee 滚动 —— 一旦内容溢出意味着设计违规，应裁剪而非滚动。
  • idle 收窄：0 活跃会话时右段隐藏，整体收窄到仅"notch + 左段 owl"， 肉眼接近"原生 notch 略微变宽"。
  • 展开（opened）允许下探，但宽度硬契约：用户 hover/click 或事件触发的 sessionList / approvalCard / questionCard / completionCard 仍按 §4 / §5 渲染为下沉面板， 下边缘可进入内容区 —— 仅 closed 态受高度约束。 展开宽度恒等于 --island-expanded-w = 480pt， 所有 variant 一致，内容不能撑大、不能压缩；超长 cap-lbl / 选项行截断 + 省略号， sessionList 行数超 5 时滚动而非加宽。 展开 ⇄ 收起的动画：宽度 296 ⇄ 480pt 横向插值 + 高度 28 ⇄ 抽屉高纵向插值， 280ms ease。
  • 非 notch 屏退化：safeAreaInsets.top == 0 时不存在物理 notch， 退化为菜单栏正下方一条同样不下探超过菜单栏底线的胶囊（保持"零下探"语义一致）。

下方 Icon / Number / Status text 三要素仍然成立， 映射到新形态：Icon = 左段彩色 owl glyph；Number = 右段首字 activeCount； Status text = 右段尾字（Error / Awaiting / Running）。形如 "2 Running"。

Icon
会换脸的猫头鹰
复用 claudedot-icons.html 的现成表情，按状态换脸： run=绿色聚焦 · wait=黄色 · err=红色怒脸 · idle=炭灰静默。 22×22 全彩渲染（内联 base64）。状态唯一表达 = glyph 颜色， 不在 owl 旁边再叠脉冲点 —— 简洁、单一信息源。
Number
activeCount
非 idle 会话总数（running + waiting + error）。 0 时折叠态收窄并隐去数字，仅留 glyph，避免噪音。
Status text
优先级最高的一行
按 error > waiting > running > idle 取最高级， 展示该级别下最具信息量的一句（见 §2）。

4 · 表面状态模型（Layout × Variant）

岛体不是「折叠 ⇄ 展开」一个维度，而是 布局（layout）× 变体（variant）两个正交维度。 §3 的优先级链只决定 closed 折叠态的文案； opened 展开态根据事件类型路由到不同变体。

路由规则。
  • manual click / hover → opened + sessionList
  • permissionRequested 事件 → opened + approvalCard（auto-expand）
  • questionAsked 事件 → opened + questionCard（auto-expand）
  • sessionCompleted 事件 → opened + completionCard（auto-expand）
auto-expand 卡片是临时表面：
  • 超时自动折回（approvalCard 12s / questionCard 12s / completionCard 6s）
  • 指针进入卡片后再离开，立即折回（即「看一眼就走」的语义）
  • 不把这些动作以 inline 形式塞进 sessionList —— 它们用独立通知表面，保证列表只负责导航
为什么这样拆？ sessionList 是「主动浏览」入口（用户在掌控）；卡片是「被动响应」入口（事件在掌控）。 把它们混在一起，会让用户每次 hover 都看到陌生卡片，反而成为干扰。

6 · 功能需求 · 交互与生命周期

Trigger
常驻显示。开关由 popover 底栏切换，状态持久化到 UserDefaults.showDynamicIsland。
Refresh
复用 AppDelegate.refresh() 的 1.5s tick，不引入新的数据源；与菜单栏猫头鹰同源。
Hover
≥ 180ms 展开为 opened · sessionList；离开后 400ms 折回。展开期间内容增量更新，不重建 NSPanel。
Click
折叠态点击 = 打开 popover；展开态点击行 = jump(pid:cwd:)。
Auto-expand
事件驱动展开为对应 variant： PreToolUse → approvalCard · UserPromptSubmit/Notification → questionCard · Stop → completionCard。 展开是动画式 push，不抢键盘焦点（becomesKey = false）， 不打断用户当前输入。
Auto-collapse
临时卡片倒计时：approvalCard / questionCard 12s，completionCard 6s。 指针进入卡片再离开 → 立即折回（即「看一眼就走」语义，避免长期遮挡）。 已经折回的事件不重复推：30s 内同一会话不重弹同类卡片。
Variant 抢占
多事件同时到达：优先级 approvalCard > questionCard > completionCard > sessionList。 高优先级到达时，低优先级卡片立即折回让位；同优先级按 FIFO 排队。
Hide
全屏应用前置时（NSMenu.menuBarVisible() == false）自动隐藏；退出全屏即归位。
Multi-display
监听 didChangeScreenParametersNotification，仅显示在内建屏，外接屏不复制。 切换时同时重算三个动态几何：menuBarHeight / realNotchWidth / 岛体居中点 （详见 §9 几何动态化）。
Non-notch Macs
safeAreaInsets.top == 0 时形态完全等同刘海机型 —— 同一颗悬浮药丸， 高度仍按 menuBarHeight − 2pt 动态算（典型 22pt），中段 fallback 220 + 24pt 安全余量。 差异仅在定位算法（§9 非刘海机型行），视觉与功能无降级。

7 · 度量与排印规范

Closed size
高度 = 菜单栏可视高度 − 2pt（运行时算，禁止硬编码）。 实测：14"/16" MBP ≈ 36pt 岛高；M1 Air ≈ 22pt 岛高；硬编码 28pt 在刘海机型上会明显矮于 status item。 下边缘绝不越过菜单栏底线。整条宽度由内容紧凑决定 —— lead 46 + notch-core 动态 + trail ~70，实测 14" ≈ 345pt（含 24pt 安全余量后的 notch 229pt）； idle 0 会话时隐藏 trail。 四角全圆角，radius = height / 2（高度变时同步，不能写死 14pt）。
Expanded size
头部 28pt + 抽屉自适应高度，宽度恒等于 --island-expanded-w = 480pt （硬契约：所有 variant 一致，min-width = max-width = 480pt， 内容不能撑大也不能压缩）；全圆角 18pt（略大于 closed 14pt）； sessionList 最多渲染 5 行、超出滚动；超长 cap-lbl / 选项文字截断 + 省略号。 仅"抽屉"部分允许下探到内容区，头部仍受 closed 高度约束。
Layout 三段
lead ~48pt：仅一个 22px 彩色 owl + 左 16 / 右 10 非对称 padding （药丸左端是 full-pill 圆弧，左 padding 必须比右 padding 大 ~6pt 才能视觉居中， 否则 owl 看起来贴着左弧很挤）。不放脉冲点（颜色即状态）。 notch-core 运行时算，不硬编码： realNotchWidth + 24pt 安全余量（左右各 12pt 缓冲）； 实测范围 14"≈229 / 16"≈249pt；无法读取时 fallback 220pt（不是 180pt）。 trail ~70pt（自适应内容、定宽不弹性）：仅两 token {activeCount} {状态词}，禁止溢出/marquee/截断。 全条总宽 = lead + 动态 notch-core + trail，因此整条胶囊在不同机型上略有差异， 这是正确行为 —— 保证 trail 永远在物理 notch 之外。
主图标
会换脸的猫头鹰 · 22×22（32pt 行高内放 22px owl + 5pt 上下气口）· 5 张表情来自 claudedot-icons.html 的 face() 配置 （run 绿 / wait 黄 / err 红怒脸 / idle 炭灰 / brand 终焰陶）。 实现层 = owlImage(for:state:) 按状态返回对应 NSImage，全彩、不染色。

8 · 非功能需求

Performance
CPU ≤ 0.5%（folded · idle）/ ≤ 1.5%（expanded）。 内容签名（signature）未变时不重建视图层，仅更新文本节点。 单次 refresh() 渲染 ≤ 8ms（与菜单栏 owl 同 tick，不引入新计时器）。
Memory
常驻 NSPanel 不复用 popover 的 NSWindow； 额外内存占用 ≤ 4 MB（含 owl glyph image cache）。
Fail-safe
数据源缺失或 SessionVM 异常时， 岛体降级为 idle（不崩、不抛），日志写入 ~/.claude/statusbar/log。 决不阻塞 popover 或菜单栏 owl 的刷新。
Accessibility
NSPanel 的 accessibilityLabel = 「Claude Dot, n active, 状态文案」。 VoiceOver 焦点可达；岛上元素不抢键盘焦点（becomesKey = false）。
Compatibility
要求 macOS 12+（依赖 safeAreaInsets）。 11 以下隐藏入口并在 popover 设置项中 disable 开关。
Privacy
展示在所有 Space / 锁屏前的 NSPanel 必须排除锁屏场景 （collectionBehavior 不含 .canJoinAllSpaces 时同 popover； 锁屏由系统兜底隐藏）。岛上不暴露完整 prompt 内容，仅显示 title / 工具名。
Theming
岛体始终 纯黑（无 light / dark 切换），与刘海无缝。 内文 accent 色随 NSApp.effectiveAppearance 提亮 ≈ 6% 以适配 dark mode 下系统壁纸偏白的情况。
Localization
状态文案模板字符串走 NSLocalizedString； 中英双语优先，主词 ≤ 12 字符，副词 ≤ 28 字符，超出截断 + 加省略号。

9 · 边界条件 · 异常处理

屏幕选择策略
不要用 NSScreen.main（= key window 所在屏，可能是外接屏）。 按下列顺序优先： safeAreaInsets.top > 0 → 非空 auxiliaryTopLeftArea / auxiliaryTopRightArea → fallback NSScreen.screens.first。 (参考 open-vibe-island/docs/island-position-investigation.md 的关键 bug)
定位数学
notch 模式从 screen.frame（不是 visibleFrame）顶部锚定， 并用 safeAreaInsets.top 作为刘海留白； fallback 模式（非刘海）从 visibleFrame.maxY 顶部居中。 用同一套 visibleFrame 公式套两种屏 → 岛体永远比刘海低 ~42pt，融不进去。
非刘海机型
safeAreaInsets.top == 0 时（M1 Air、Intel Mac 等）形态完全等同刘海机型 —— 同样的 28pt 全圆角悬浮胶囊，centered 在菜单栏内。差异仅在定位算法： notch 模式从 screen.frame.top + safeAreaInsets.top 起算的上气口； fallback 从 screen.frame.top + 2pt 直接定位。 视觉与功能无降级。
全屏应用
监听 NSMenu.menuBarVisible()； 全屏 IDE / 视频 / 演示前置时 orderOut， 退出全屏后下个 tick 自动 orderFrontRegardless。
外接显示器
仅在被选中的 built-in 屏显示，外接屏不复制； didChangeScreenParameters 触发时重跑选择策略 + 重计 frame。
Bridge fail-open
岛体异常时 绝不阻塞 hook。 hook 写状态文件即返回，岛体读不到也只是退化为 idle； 数据源缺失 → 仍显示炭灰 owl +「Claude Dot」（idle 脸），不显示 error 红脸 （error 是 agent 错误，不是岛体错误）。
通知声音
completionCard 默认静音（环境层不应制造听觉打扰）； approvalCard / questionCard 可在偏好里开启系统提示音（默认关）。 mute 开关与 popover 共享一个 UserDefaults.soundsMuted。
同名会话
展开列表中两个会话 title 相同时，副词追加 cwd 末段（…/repo-name）做区分。
超 5 个会话
展开列表最多 5 行，溢出底部出现 “+n more · open popover” 行，点击打开 popover。
error 衰减
error_at 距今 > 90s 后，岛体回落到次高优先级。 若所有 error 都已衰减且无 running / waiting，回到 idle 收窄态。
用户主动关闭
UserDefaults.showDynamicIsland = false 时立即 orderOut， 下次启动也不显示；偏好与菜单栏 owl 完全独立。
菜单栏自动隐藏
系统偏好 "Automatically hide and show the menu bar" 开启时菜单栏会上滑消失。 监听 NSWorkspace.shared.notificationCenter 的菜单栏可见性通知 + NSStatusBar.isVisible；菜单栏不可见时 orderOut， 重现时下一 tick 自动 orderFrontRegardless。不跟随菜单栏滑动动画 —— 直接隐/显，避免追帧抖动。
多 Space / 桌面切换
NSPanel 的 collectionBehavior = [.canJoinAllSpaces, .stationary, .ignoresCycle] —— 岛体在所有 Space 都可见，但不参与 ⌘Tab 循环。 Mission Control 激活时 macOS 会自动隐藏 status item 类窗口；岛体跟随系统行为，不主动干预。
Stage Manager / 全屏沉浸
Stage Manager 激活时左侧卡片区可能与岛体重叠。岛体 level = .statusBar 保证位于其上层；Stage Manager 自动隐藏菜单栏时遵循上面的"菜单栏自动隐藏"行为。 视频全屏 / Keynote 演示前置时复用"全屏应用"分支 orderOut。
Game Mode（macOS 14+）
系统 Game Mode 开启时 macOS 自动抢占菜单栏 + 隐藏所有 status item。 监听 NSWorkspace.didActivateApplicationNotification 中 game-mode 进程； 检测到时 orderOut，退出后恢复。不尝试覆盖系统的优先级（无意义且违规）。
count 上限
activeCount ≥ 100 时显示 "99+"（serif，保持 tabular 等宽）。 理论极限：trail 区禁止溢出（§7），所以 count token 字符数 ≤ 3 是硬约束。 实际操作系统中同时跑 100+ 个 Claude session 极少见，但需要兜底防排版破相。
状态文件并发写
多 session 同时触发 hook → 并发写 ~/.claude/statusbar/sessions/<sid>.json。 必须 atomic：写到 <sid>.json.tmp.<pid> + rename(2)。 岛体的 reader 用 O_NONBLOCK 读，遇到 EOF/truncated 当作"上一帧仍有效"忽略本次， 不显示半边状态。
时钟基准（90s error 衰减）
NTP 校时、夏令时切换、用户手动改时间都可能让 Date() 跳变 ± 小时级别。 error_at 衰减必须用 monotonic 时钟 （CFAbsoluteTime 或 mach_absolute_time），不用 wall clock， 否则一次校时可能把红脸卡死或瞬间过期。hook 端写入 error_at 时同时记一份 error_at_mono 单调时间，reader 用单调差值判定衰减。
进程僵尸 / 失活
kill(pid, 0) == 0 仅说明 pid 表项存在，可能是 zombie / SIGSTOP'd。 补充判定：ps -o stat= -p <pid> 状态字符含 Z（zombie）或 连续 3 tick 文件 mtime 不变且 status == busy → 视为失活，从聚合移除。
状态翻转防抖
单 session 在 PreToolUse → PostToolUse 之间会快速从 running → waiting → running 翻转， 肉眼看就是闪烁。聚合状态升级（→ waiting）立即生效（不阻塞用户响应）， 降级（waiting → running / running → idle）需稳定 600ms 才生效， 通过 SessionVM 一层 debounce 实现。
冷启动批量 sessions
app 启动时 ~/.claude/sessions/ 可能已有 20+ 个 pid 文件。 首帧同步扫一遍（≤ 50ms），先以聚合状态绘制；后台异步算逐 session token 等次要数据。 决不让 UI 出现 200ms 以上的 "0 sessions" 错误中间态。
Owl glyph 加载失败
打包的 5 张 PNG 缺失或解码失败时，fallback 到 Status.emoji （headless 也用的同套）：⚪ idle · 🟢 run · 🟡 wait · 🔴 err。 字符渲染优于空白圆角矩形；不抛 assertion。
运行时主题切换
岛体始终纯黑，不跟随 NSApp.effectiveAppearance 切换。 仅展开抽屉里的 sessionList / cards 使用 Theme.dark 一套色， 系统外观切换时只重绘抽屉、不动药丸本身，避免黑色变灰这种诡异过渡。
Reduced Motion / Increased Contrast
NSWorkspace.shared.accessibilityDisplayShouldReduceMotion == true 时 关闭展开/收回弹性动画（直接淡入淡出 120ms）。 shouldIncreaseContrast 时把 ink-2 / ink-3 提到 ink-1，状态词加粗到 600。
物理 notch 几何兜底
safeAreaInsets.top 在某些 macOS 升级 / 外接屏断连瞬间可能返回 0 或错值。 缓存上次成功的 inset 值；连续 5 tick 异常时回退到"非刘海机型"分支，并写一条 warning 到 ~/.claude/statusbar/log。
菜单栏高度差异（实机 bug）
已观测 bug：硬编码 islandHeight = 28pt 时，14"/16" MBP 上菜单栏实际 ≈ 38pt → 岛体明显矮于旁边的微信、电池等 status item，视觉违和（截图证据：胶囊上下各空一截）。 修法：岛高必须从 NSScreen 推导：
1) safeAreaInsets.top > 0（刘海机型）→ menuBarHeight = safeAreaInsets.top（含刘海留白即菜单栏高）
2) frame.height − visibleFrame.height（非刘海机型，dock 在底时即菜单栏高）
3) 兜底 NSStatusBar.system.thickness + 2 ≈ 24pt
islandHeight = menuBarHeight − 2pt（上下各 1pt 气口）， islandRadius = islandHeight / 2（同步重算）。 NSWorkspace.didChangeScreenParametersNotification 触发时重算。 验收：岛体与旁边 status item 视觉等高，胶囊圆度保持 full-pill（高度变时圆角同步）。
notch 宽度差异（实机 bug）
已观测 bug：硬编码 notchCoreWidth = 180pt 时，14"/16" MBP 上物理 notch 实际占 205-225pt → trail 首 token（数字 {count}）落入物理 notch 范围被遮挡， 肉眼看就是 "5 Awaiting" 只剩 "Awaiting"。 修法：notch-core 宽度必须从 NSScreen.auxiliaryTopLeftArea / auxiliaryTopRightArea 推导 （= 屏宽 − 左侧菜单可用区右边界 − 右侧状态项区左边界）， 再左右各加 12pt 安全余量（避开 notch 的边缘半透明渐变）。 无 API 可用时 fallback = 220pt（覆盖最宽 16" notch）， 不是 180pt。 验收：14"/16" MBP 上 trail 的 count 字符完全可见、不被 notch 蹭到任何像素。
隐私 — 屏幕录制 / 投屏
CGDisplayStream 活跃（被录屏 / AirPlay / 投屏）时不主动隐藏， 但右段状态词降级到通用 "Active"（不暴露 Error/Awaiting/Running 这种 可能透露工作内容的词），sessionList 展开时不显示 cwd 与具体 prompt。 这是 §8 Privacy 行的运行时补充。
磁盘满 / HOME 只读
hook 写入失败时静默丢弃本次状态（exit 0），不阻塞 Claude。 岛体读取失败（EACCES / ENOSPC）时退化为 idle 并显示 ink-3 toast 一次："状态目录不可写"，链接到 popover 设置 → 重选目录或禁用。
触控板 / 系统手势透传
岛体 NSPanel ignoresMouseEvents 区域 = lead + notch-core 上半 14pt （上半属于"伪 notch"，传统上 macOS 上滑手势触发 Mission Control 应该可用）。 仅 trail + 抽屉处理鼠标事件。三指上滑等手势不被岛体吃掉。
非目标（再次明确）
❌ 不在岛上代答 permission（仅渲染 + jump） ❌ 不展示 token / subscription ❌ 不替代菜单栏 owl ❌ 不复制到外接显示器 ❌ 不支持锁屏 widget ❌ 不接收键盘输入 ❌ 不引入网络栈（Watch / Bonjour 留独立模块）
